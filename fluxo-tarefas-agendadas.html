Bugs Encontrados e Status

🔴 BUG #1 — crm:'esporte' vs 'esportes'  ✓ CORRIGIDO

Onde: prospeccao-diaria-03h — PASSO 8 — mapeamento de segmentos
Problema: O campo crm para esportes era "esporte" (sem 's') no daily-sync. Todas as outras tarefas (revisões 12h, 16h, sexta-16h e follow-up) usavam "esportes". Os leads criados pela prospecção ficavam com um valor de CRM diferente dos atualizados pelas análises — podendo gerar registros duplicados ou leads "perdidos" no funil.
Correção: Alterado para "esportes" em todas as ocorrências do mapeamento.

🔴 BUG #2 — fs.readFileSync no browser (Unimidia Robô 4)  ✓ CORRIGIDO

Onde: unimidia-robo4-relatorio-semanal — Passo 4 — envio do PDF por e-mail
Problema: O código usava require('fs') e fs.readFileSync() dentro do javascript_tool (contexto browser/Chrome), onde APIs Node.js não existem. O envio do PDF anexado sempre falharia.
Correção: O PDF agora é lido via bash (Python + base64) primeiro, e o conteúdo base64 resultante é passado manualmente para o javascript_tool que faz o POST do e-mail.

🟡 AVISO — Nomes das tarefas vs horário real de execução

revisao-whatsapp-15h executa às 12h (cron 0 12 * * 1-4) mas salva os arquivos como _15h.json.
revisao-whatsapp-18h executa às 16h (cron 0 16 * * 1-4) mas salva os arquivos como _18h.json.
O fluxo funciona corretamente porque a nomenclatura dos arquivos é consistente internamente — mas os nomes das tarefas são confusos. Não requer correção urgente.
